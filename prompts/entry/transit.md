# Transit reading — entry fragment

Applies when `reading_type` is `transit`. Append this framing to
`prompts/entry/_reading.md`. This fragment adds **framing only**; it never
calculates transits, stations, or orbs.

## Chart-data emphasis

- `chart_data.timing_factors[]` entries where `technique` is `transit`:
  `triggering_body`, `natal_body`, `aspect`, `orb_degrees`, and `date`.
- The **natal baseline** (Ascendant, sect, natal placements) that the transits
  act upon.

## Framing for the reading

- Treat each supplied transit as the primary timing testimony; do not derive
  missing transits, station dates, retrograde passes, or house contacts.
- Use dignity/debility/condition only when the source explicitly supplies it,
  or label the language as general sign symbolism.
- Load `references/reading_types/transit.md`; load
  `references/reading_types/transit_examples.md` when a supplied transit
  matches its trigger set (Saturn→Moon, Jupiter→Sun, Mars→Ascendant,
  Pluto→angle). Selection is owned by `SKILL.md`; this fragment only names it.
